# 自动Git提交规则

## 规则描述
当回答完用户的问题后，如果对文件进行了修改，应自动提交到git仓库。

## 适用场景
- 代码修改
- 配置文件更新
- 资源文件变更

## 执行步骤
1. 检测是否有文件被修改
2. 如果有修改，执行以下操作：
   - `git add .` - 添加所有修改的文件
   - `git commit -m "自动提交: 回答用户问题后的修改"` - 提交修改
   - `git push origin main` - 推送到远程仓库

## 注意事项
- 只在确实有文件修改时执行提交
- 确保提交信息清晰明了
- 遵循项目的git工作流程

## 实现方式
可以在每次完成任务后检查文件变更并执行提交操作。
