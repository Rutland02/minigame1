# 三色融澄·数字赋能：海澄村游戏化小程序项目计划书

## 一、项目概述

### 1.1 项目名称
三色融澄·数字赋能——海澄村非遗/自然/红色资源游戏化传播小程序

### 1.2 项目背景
海澄村拥有丰富的非遗、自然和红色资源，但这些资源的传播方式较为传统，缺乏互动性和趣味性。本项目旨在通过游戏化的方式，让更多人特别是年轻一代了解和关注海澄村的特色资源，同时为大创项目提供可量化的传播数据支撑。

### 1.3 核心目标
- 以轻量游戏化形式，实现海澄村非遗、自然、红色三色资源的趣味化传播与互动式教育
- 为大创项目提供可量化的传播数据支撑
- 提升海澄村的知名度和文化影响力
- 促进海澄村的文旅融合发展

### 1.4 技术选型
- 微信小游戏模板
- 轻量游戏化框架（消消乐/拼图/答题）
- 本地存储（数据存储）
- 前端逻辑处理

## 二、核心功能模块

### 2.1 三色消消乐（核心玩法）
- **玩法逻辑**：6×6或8×8的方块矩阵，方块元素为海澄村三色资源的图标，通过滑动相邻方块使3个及以上相同元素连成一线即可消除，获得对应积分
- **游戏规则**：每局限时60秒，根据消除数量和连击数结算积分，积分可用于解锁成就
- **难度梯度**：
  - 初级：仅出现2种资源元素，方块移动速度慢
  - 中级：出现3种资源元素，增加特殊方块（如“一键消除整行”）
  - 高级：限时缩短至45秒，增加干扰方块（如“冻结5秒”）

### 2.2 三色拼图（辅助玩法）
- **玩法逻辑**：提供海澄村古树、红色遗址、非遗作品的高清图片，切割为9/16/25块拼图，玩家拖动碎片完成拼图
- **特色设计**：拼图完成后，自动生成“打卡海报”，可分享至朋友圈，附带“我在海澄村解锁了XX资源”的文案
- **解锁内容**：完成拼图后解锁对应资源的科普文案和音频故事

### 2.3 三色答题（教育玩法）
- **题库分类**：
  - 非遗题：三灶鹤舞的历史、竹编的工艺、非遗传承人的故事
  - 自然题：古树的种类、生态保护的知识、海澄村的自然景观
  - 红色题：海澄村的革命历史、红色遗址的故事、红色精神的内涵
- **答题规则**：每次随机抽取5道题，答对1题得10积分，答错不扣分
- **特殊机制**：连续答对3题可获得“知识达人”buff，下一局消消乐积分翻倍

### 2.4 线下打卡系统
- **功能描述**：用户到海澄村实地打卡，扫描指定二维码解锁专属皮肤和彩蛋
- **技术实现**：基于微信小程序的地理位置和扫码功能

### 2.5 成就系统
- **成就分类**：
  - 非遗类：完成10次非遗元素消除、解锁全部非遗拼图、答对全部非遗答题
  - 自然类：完成20次自然元素消除、解锁全部古树拼图、答对全部自然答题
  - 红色类：完成15次红色元素消除、解锁全部红色遗址拼图、答对全部红色答题
- **成就奖励**：解锁专属头像框、游戏皮肤，生成“海澄村数字体验证书”

## 三、页面设计

### 3.1 登录页
- **核心元素**：
  - 顶部：项目Logo（“三色融澄”+海澄村剪影）
  - 中部：微信授权登录按钮（“一键登录，开启海澄村数字之旅”）
  - 底部：项目简介（“非遗·自然·红色，在游戏中读懂海澄村”）

### 3.2 首页
- **顶部导航**：
  - 左侧：用户头像、昵称、当前等级（如“古树守护者Lv.2”）
  - 右侧：积分余额、消息通知（如新成就解锁）
- **中部功能区**：
  - 游戏入口卡片：
    - 「三色消消乐」：图标为彩色方块，附带“消除资源，解锁知识”文案
    - 「三色拼图」：图标为拼图碎片，附带“拼出海澄，发现美好”文案
    - 「三色答题」：图标为问号，附带“答题闯关，成为知识达人”文案
  - 线下打卡入口：
    - 「扫码解锁彩蛋」：图标为二维码，附带“到海澄村实地打卡，解锁专属皮肤”文案
- **底部成就栏**：展示已解锁的3个成就图标（如“非遗小达人”“古树守护者”“红色传承人”），点击可查看成就详情

### 3.3 游戏页

#### 3.3.1 三色消消乐游戏页
- **顶部**：倒计时（60秒）、当前积分、连击数
- **中部**：6×6方块矩阵，元素为三色资源图标
- **底部**：
  - 道具栏：特殊方块（如“一键消除”“冻结时间”），消耗积分使用
  - 结算按钮：提前结束游戏，查看本局成绩

#### 3.3.2 三色拼图游戏页
- **顶部**：拼图名称（如“海澄村古树拼图”）、剩余时间
- **中部**：拼图区域，展示当前拼图碎片和目标图案预览
- **底部**：
  - 难度选择：9/16/25块拼图切换
  - 提示按钮：消耗积分获得拼图提示
  - 重置按钮：重新开始当前拼图
  - 完成后：生成打卡海报分享按钮

#### 3.3.3 三色答题游戏页
- **顶部**：当前题目序号（如“第3题/共5题”）、剩余时间
- **中部**：题目内容、选项（A/B/C/D）
- **底部**：
  - 提交按钮：提交当前答案
  - 提示按钮：消耗积分获得题目提示
  - 完成后：显示答题结果和解析

### 3.4 成就页
- **成就分类**：按非遗、自然、红色三类展示成就
- **成就详情**：每个成就的解锁条件、当前进度、解锁奖励
- **证书生成**：达到一定成就后生成“海澄村数字体验证书”

## 四、技术架构

### 4.1 前端技术
- 微信小游戏框架：基于微信小游戏模板开发
- 游戏引擎：使用Canvas API实现游戏逻辑
- UI组件：使用微信小游戏原生组件
- 动画效果：使用Canvas动画和帧动画

### 4.2 数据存储
- 本地存储：使用wx.setStorageSync和wx.getStorageSync存储用户数据
- 数据结构：使用JSON格式存储游戏数据、成就数据

### 4.3 数据结构
- **用户数据**：用户基本信息、积分、等级、成就进度
- **游戏数据**：游戏记录、得分、连击数
- **题库数据**：题目内容、选项、答案、分类
- **资源数据**：海澄村资源信息、图片、音频、科普文案

## 八、风险评估与应对策略

### 8.1 风险评估
- **技术风险**：游戏逻辑复杂度较高，可能存在性能问题
- **内容风险**：海澄村资源资料收集不完整，可能影响游戏内容质量
- **运营风险**：用户获取难度较大，可能影响项目推广效果

### 8.2 应对策略
- **技术风险**：采用模块化设计，优化游戏算法，进行性能测试和优化
- **内容风险**：与海澄村当地政府、文化部门合作，确保资源资料的完整性和准确性
- **运营风险**：制定合理的推广策略，利用微信生态进行传播，开展线上线下活动
